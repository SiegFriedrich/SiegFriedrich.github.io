JSON&AJAX基础

发表于 2019-11-20 更新于 2019-11-25 分类于 前端 阅读次数：

Json和Ajax无论作为前端还是后端,都是基础中的基础,实际开发中经常需要用到的概念.这次好好的再将基础重新拿出来在系统的学习学习.

Ajax = Asynchronous Javascript and XML;

异步的JavaScript 和 XML 技术, 页面无刷新.

优点:

异步模式, 提升用户体验.
优化浏览器,服务器之间的传输,减少不必要的数据往返,减少贷款占用.
客户端运行ajax,承担一部分由服务器承担的工作,从而减少大用户量的服务负载.

缺点:

不支持页面的back回退按钮
安全问题 暴露了与服务器交互的细节
对搜索引擎支持比较弱

Ajax

XMLHttpRequest对象

支持异步请求的技术,他是Ajax的核心
可以向服务器提出请求并处理响应,而不阻塞用户
可以在页面加载以后进行页面的局部更新

  • 如何使用AJAX?

1)创建XMLHttpRequest对象,也就是创建一个异步调用对象

2)创建一个新的HTTP请求,并指定该HTTP请求的方法,URL

3)设置响应HTTP请求状态变化的函数

创建XMLHttpRequest对象

var XMLR = new XMLHttpRequest();

open方法

语法 open(method, url, asyn);
功能 创建HTTP请求,规定请求的类型,URL及是否异步处理请求

method: GET / POST
URL: 文件在服务器上的位置
async : true(异步) / false(同步)

open方法不是向服务器发送真正的请求,只是初始化请求并发送相同域相同端口的URL的请求,否则因为安全问题会报错.

url是必须填写的参数,可以使.txt .xml 也可以是.asp 和 .php
(在回传响应之前,能够在服务器上执行任务)

同步和异步的区别
同步:提交请求–>提交服务器处理–>处理完毕返回,这个期间客户端浏览器不能干恩和事情
异步:请求通过事件触发–>服务器处理(此时,浏览器可以做其他事情)–>处理完毕

async:布尔值 默认是true 异步的.

设置状态请求变化函数 与 send

设置HTTP头

像HTML表单那样的POST数据, setRequestHeader()来添加HTTP头,然后在send方法中规定希望发送的数据;

语法:xmlr.setRequestHeader(heaer,value);

使用:xmlr.setRequestHeader(‘ContentType’,’application/x-www-form-urlencoded’);

获取服务器端返回的数据

在获取到响应后,响应的数据会填充到XHR对象的属性,有四个相关属性会被填充:
responseText – 从服务器进程返回数据的字符串形式
responseXML – 从服务器进程返回DOM兼容的文档数据对象
status – 从服务器返回的数字代码,如404,200
status Text – 伴随状态码的字符串信息

Json

Json I

JavaScript Object Notation JavaScript对象表示法,它是一种数据交换的文本格式,而不是一种编程语言,用于读取结构化数据,2001年由Douglas Crockford提出,用于替代繁琐的XML格式

支持三种数据格式
I 简单值 字符串(必须使用双引号),数值(必须是十进制),布尔值,null 不能使用NaN,Infinity,不支持undefined

II 对象 不应该出现两个相同的属性,以键值对的方式表现出来,键名必须使用双引号

III 数组 可以通过数值索引来访问其中的值,数组最后一个成员后面不能加逗号

JSONP跨域

什么是跨域

  • 同源策略 : 域名,协议,端口均相同.
  • 跨域 : 上述三点有一点不同,被称作跨域.
  • 如何解决跨域
    • 跨域资源共享(CORS)
    • 使用JSON(常用)
    • 修改document.domain
    • 使用window.name

什么是JSONP?
JSON with padding(填充式json)的简写,是应用JSON的一种新方法,也是一种跨域的请求方案.

JSONP的组成
两部分组成: 回调函数和数据.回调函数:当响应到来时应该在页面中调用的函数,而数据就是传入回调函数中的JSON数据

JSONP的原理
直接使用XMLHttpRequest请求不同域的数据是不可以的,但是在页面上引入不同域上的js脚本文件就可以,JSONP正是利用这一特性来实现的
